# BM TOOLS 商业化开发变更说明

本文档用于记录 BM TOOLS 本轮面向商业运营版本的功能开发、结构调整和体验优化，方便后续继续迭代、测试、交付和对外说明。

## 一、新增功能

### 0.1 商业运维稳定性增强

本轮继续补强监控模块的商业运营细节。

新增：

- 监控表结构自动升级服务 `DatabaseUpgradeService`
- 前台 `/status` 访问时自动兼容旧库字段
- 后台监控管理进入时自动补齐 SSH / HTTP 探针字段
- 监控来源标签：宝塔 / SSH / 探针
- HTTP 探针 Token 一键生成
- HTTP 探针部署助手和复制部署说明
- 后台监控概览卡片：总数、在线、异常、探针接入
- 状态页数据新鲜度提示：刚刚更新、几分钟前、数据过期
- 状态页读取失败时显示明确提示，不再空白

商业价值：

- 旧版本升级新监控能力时更顺滑。
- 用户不用手动执行 SQL 才能看到页面。
- 运营后台更清楚每台服务器使用哪种监控方式。
- 减少“页面空白”“字段不存在”“不知道填什么 Token”等低级售后问题。
- 降低 HTTP 探针接入门槛，让普通运营人员也能按步骤接入服务器。
- 防止用户误把过期监控数据当作实时状态。

### 0.2 用户工作台商业体验增强

本轮继续优化用户高频使用页面。

新增：

- 图床页面增加套餐、容量、剩余空间、图片数量、单文件限制展示
- 图床容量接近上限时显示升级提示
- 图床上传区增加上传规则说明和更清晰的空状态
- 网盘页面增加套餐、容量、文件/文件夹数量、单文件限制展示
- 网盘容量接近上限时显示扩容提示
- 网盘页面增加文件工作台说明、空文件夹/空文件提示
- 用户中心重构为账户工作台
- 用户中心增加存储进度、流量进度、文件资产、余额状态
- 用户中心增加“下一步建议”：充值、扩容、上传、整理网盘
- 用户中心增加最近文件列表

商业价值：

- 用户进入页面后知道自己还能用多少容量、当前套餐有什么限制。
- 上传失败前提前提示容量风险，减少售后。
- 用户中心从静态统计升级为可行动的工作台。
- 更符合现代 SaaS 产品“状态 + 建议 + 快捷操作”的使用逻辑。

### 0. 白蓝商业云控制台视觉体系

本轮将前台、后台和安装向导统一调整为 BM TOOLS 白色 + 蓝色的商业云控制台风格。

新增视觉规范：

- 左侧产品导航栏
- 顶部操作栏
- 白蓝毛玻璃面板
- 轻量阴影层级
- 统一 8px 圆角
- 蓝色主按钮
- 图标化导航与按钮
- 表格、表单、卡片统一为云产品后台风格
- 支持自动昼夜模式、浅色模式、深色模式

商业价值：

- 更像可对外交付的云工具平台，不再像临时模板。
- 后台管理长期使用更清晰，运营人员能快速找到用户、套餐、订单、存储、监控等入口。
- 安装页、前台页、后台页视觉统一，有利于品牌化运营。

### 1. 首页模板风格切换

后台系统设置新增“首页模板风格”选项，可切换不同视觉风格的前台首页。

当前包含：

- 玻璃上传风
- 星云科技风
- 极简留白风
- 创作者画廊风
- 极光卡片风
- 开发者终端风

商业价值：

- 方便运营方根据目标用户群体调整前台气质。
- 适合做不同品牌定位，如开发者工具站、创作者素材站、团队网盘站。
- 后续可继续扩展节日活动模板、企业模板、极简备案模板。

### 2. 自动昼夜主题模式

新增 `auto` 主题模式，系统可根据访问者本地时间自动切换浅色或深色。

包含：

- 后台默认主题支持“自动跟随昼夜”
- 前台主题按钮支持“自动 / 浅色 / 深色”循环切换
- 用户偏好保存在浏览器本地
- 后台管理界面也支持主题切换

商业价值：

- 降低夜间访问刺眼感。
- 提升现代产品体验。
- 适合长期运营型工具平台。

### 3. 运营方服务器公开状态页

服务器监控调整为运营方维护，前台仅展示公开状态，不再让普通用户添加自己的服务器。

新增能力：

- 前台 `/status` 展示运营方服务器状态
- 后台添加运营服务器
- 后台测试宝塔 API 连接
- 后台立即采集服务器状态
- 支持 CPU、内存、磁盘、服务状态和告警
- 前台不暴露宝塔地址和 API 密钥

商业价值：

- 用户可以看到平台运行状态，增强信任感。
- 运维信息由管理员统一掌控，避免普通用户误用监控模块。
- 可作为平台可靠性展示页面。

### 4. 后台本机服务器环境状态

后台首页新增本机服务器和运行环境信息。

展示内容：

- 操作系统
- Web Server
- PHP 版本
- MySQL 版本
- 当前服务器时间
- PHP 内存限制
- 上传大小限制
- POST 大小限制
- 最大执行时间
- 磁盘总量
- 磁盘剩余
- 关键扩展检测：PDO、curl、openssl、fileinfo、mbstring、gd

商业价值：

- 管理员进入后台即可判断部署环境是否健康。
- 便于售后排障和宝塔环境检查。
- 降低因为 PHP 扩展、上传限制、磁盘空间导致的运营事故。

### 5. 图床图片详情页

新增图片详情页。

包含：

- 原图预览
- 原图直链
- Markdown 链接
- HTML 链接
- BBCode 链接
- 一键复制全部链接
- 图片尺寸
- 文件大小
- MIME 类型
- MD5
- 存储路径

商业价值：

- 图床用户可以更快复制不同场景需要的链接。
- 适配博客、论坛、Markdown 编辑器、文档站等典型用户场景。
- 提升图床产品的专业度。

### 6. 图床与网盘回收站

删除图片或网盘文件时不再直接删除远程对象，而是先进入回收站。

新增能力：

- 图床图片移入回收站
- 网盘文件移入回收站
- 回收站列表
- 文件恢复
- 文件彻底删除
- 彻底删除时删除远程对象

商业价值：

- 减少用户误删造成的损失。
- 降低客服处理压力。
- 更符合商业云存储产品的安全预期。

### 7. 网盘批量删除

网盘文件支持批量选择并移入回收站。

商业价值：

- 提升文件整理效率。
- 对内容创作者、小团队用户更实用。

### 8. 网盘文件夹管理增强

新增文件夹删除能力。

当前策略：

- 支持删除不含子文件夹的文件夹
- 文件夹内直接文件会移入回收站
- 如果包含子文件夹，会提示先处理子文件夹

商业价值：

- 保守安全，降低误删整棵目录树的风险。
- 后续可扩展递归删除确认页。

### 9. 分享下载次数限制

网盘分享表单增加下载次数限制字段。

商业价值：

- 支持临时资源分发。
- 更适合商业资料、课程文件、私密文件分享。

### 10. 下一步式安装向导

安装页面从单屏长表单调整为商业软件常见的分步安装向导。

新增流程：

- 第一步：环境检测
- 第二步：数据库配置
- 第三步：管理员配置
- 第四步：确认安装

体验增强：

- 顶部步骤进度条
- 上一步 / 下一步切换
- 当前步骤必填校验
- 最终安装信息汇总
- 页面内错误提示
- 安装按钮原生提交兜底
- 后端补充安装确认校验

商业价值：

- 降低宝塔部署门槛。
- 避免首次安装页面显得粗糙。
- 安装失败时能更清晰地提示用户修复方向。

### 11. 安装前数据库连接测试

安装向导新增数据库测试接口：

- `/install/test-db`
- 使用当前表单中的数据库地址、端口、库名、用户名和密码测试连接
- 测试成功后才允许进入最终安装
- 1045、1049、2002、1142 等常见 MySQL 错误会转为中文排障提示

商业价值：

- 避免用户点击“开始安装”后无反馈。
- 宝塔环境下能明确提示不要盲目使用 root，需要使用数据库页面中的专用账号。
- 降低首次部署失败率。

### 12. 安装完成自动进入后台

安装成功后会：

- 写入数据库配置文件
- 创建安装锁
- 创建管理员账号
- 自动写入管理员登录态
- 直接跳转后台首页

商业价值：

- 安装完成后的路径更短，符合商业软件“一装即用”的体验。
- 避免新用户装完后不知道后台入口。

## 二、修改内容

### 1. 系统品牌改为 BM TOOLS

默认项目名称从 SkyStorage 调整为 BM TOOLS。

涉及：

- 默认配置名称
- 安装向导标题
- 默认安装站点名称
- 布局默认站点名
- README 标题
- 邮件测试默认文案
- 支付订单默认站点名称
- 监控与抓取 User-Agent

保留能力：

- 安装时仍可自定义网站名称
- 后台系统设置仍可修改网站名称

### 2. 默认数据表前缀调整

安装向导默认表前缀由 `ss_` 调整为 `bm_`。

商业价值：

- 与 BM TOOLS 品牌统一。
- 新安装站点数据库辨识度更高。

### 3. 服务器监控定位调整

原先普通用户可管理服务器监控，现调整为运营方后台维护。

调整后：

- 普通用户访问 `/monitor` 会跳转公开状态页
- 管理员在后台“监控管理”维护服务器
- 前台只展示公开健康状态

商业价值：

- 产品定位更清晰。
- 个人中心聚焦图床和网盘服务。
- 监控能力成为平台可靠性展示，而不是用户套餐功能。

### 4. 套餐页面文案调整

套餐页面移除“支持服务器监控”的用户套餐表达。

现在重点展示：

- 图床
- 网盘
- API
- 分享
- 容量
- 流量
- 单文件限制

商业价值：

- 避免用户误解监控是个人服务。
- 让套餐权益更聚焦核心付费点。

### 5. 删除逻辑调整

文件删除从“立即删除数据库记录和远程对象”调整为“先进入回收站”。

影响：

- 用户删除图片后可以恢复
- 用户删除网盘文件后可以恢复
- 彻底删除才删除远程对象

商业价值：

- 更符合商业云盘安全体验。
- 降低误操作风险。

## 三、删除内容

### 2. 删除普通用户监控入口定位

普通用户导航不再提供服务器监控管理入口。

保留：

- 前台状态页
- 后台监控管理

商业价值：

- 用户中心更简洁。
- 避免低频高风险功能干扰核心业务。

## 2026-05-11 服务器状态页重构

### 新增
- `/status` 改为独立的运营方服务器状态页，不再依赖全局前台样式。
- 增加蓝白运营控制台布局、顶部栏、状态卡片、服务器列表、硬盘进度、网络速率和状态标签。
- 点击服务器后打开详情抽屉，展示 CPU、内存、硬盘、负载、上传下载、服务状态和趋势图。
- 趋势图使用内置 Canvas 绘制，避免外部 JS 加载失败导致页面空白。

### 修改
- 服务器状态页只展示运营方在后台维护的监控节点。
- 硬盘信息过滤 `/run`、`/dev`、`/sys`、`/proc`、Docker overlay、tmpfs 等无运营意义的挂载点。
- 页面所有可见文本统一为正常 UTF-8 中文，移除不稳定符号，避免字体缺失造成乱码感。
- 详情数据写入 HTML 属性时增加 JSON_HEX 安全标记，降低特殊字符破坏页面的风险。

### 优化
- 夜间模式按钮宽度修复，避免文字被挤压。
