# BM TOOLS

BM TOOLS 是一套基于 **PHP 7.4 + MySQL 5.7** 开发的综合型图床、轻量网盘、充值套餐、多对象存储和服务器状态展示系统。

当前版本：**1.0.0**

开源发布与商标保留说明见 [NOTICE.md](NOTICE.md)。

项目面向个人站长、开发者、内容创作者和小团队，适合部署在宝塔面板环境中，用于学习研究、二次开发、个人自建站点和合法商业化运营。

> 本项目仅用于正规学习研究、正向开发教学和合法业务系统开发，不提供任何隐藏后门、未授权控制或破坏性能力。

## 功能模块

- 用户系统：注册、登录、找回密码、资料修改、API Token、登录日志、权限隔离。
- 图床服务：图片上传、批量上传、拖拽上传、粘贴上传、URL 拉取、图片管理、外链生成。
- 轻量网盘：文件上传、文件夹管理、重命名、移动、删除、回收站、分享链接、提取码。
- 个人仓库：用户可创建独立仓库链接，自定义访问路径，设置公开、密码访问或私密关闭，按文件控制公开/私密，并查看访问日志和下载日志。
- 套餐充值：套餐购买、续费、升级、余额支付、余额卡密充值、套餐卡密兑换、充值记录、订单记录、余额流水。
- 支付系统：余额支付、人工充值、支付宝当面付，预留微信支付、易支付扩展结构。
- 商业卡密：后台生成余额充值卡和套餐兑换卡，支持批次、渠道、过期时间、停用、核销记录和用户自助兑换。
- 对象存储：本地存储、阿里云 OSS、腾讯云 COS、七牛云 Kodo、S3 兼容存储、MinIO、Cloudflare R2、WebDAV。
- 多存储容灾：支持上传时同步备份到多个存储源，支持失败任务重试、历史补备和存储迁移。
- 同系统对接：一套 BM TOOLS 可以通过 API 把另一套 BM TOOLS 当作图床/网盘仓储节点。
- 服务器监控：宝塔 API、SSH、HTTP 探针三种采集方式，前台公开状态展示。
- 公告通知：平台公告、系统公告、帮助教程、站内通知、邮件通知、企业微信/钉钉/Telegram 告警外送。
- API 开放：上传文件、文件列表、删除文件、容量查询，支持按应用创建独立 Token、权限范围和 IP 白名单。
- 后台管理：用户、套餐、订单、充值、支付、存储、文件、监控、公告、系统设置、管理员日志。
- 安装向导：通过 `/install` 完成环境检测、数据库配置、管理员创建和初始化安装。

## 安装部署

### 环境要求

- PHP 7.4
- MySQL 5.7+
- PDO
- curl
- openssl
- fileinfo

- gd
- ZipArchive（在线更新安装 ZIP 包时需要）

### 宝塔安装

1. 在宝塔中新建站点，PHP 版本选择 7.4。
2. 创建 MySQL 数据库和数据库用户。
3. 上传项目文件到站点目录。
4. 推荐将网站运行目录设置为 `public`。
5. 确认以下目录可写：

```text
config
storage
storage/logs
storage/cache
public/uploads
```

6. 访问安装地址：

```text
https://你的域名/install
```

7. 按页面提示完成安装。

安装完成后会生成：

```text
storage/install.lock
```

系统会根据安装锁阻止重复安装。

### Nginx 伪静态

项目提供宝塔 Nginx 伪静态参考：

```text
bt-nginx-rewrite.conf
```

Apache 环境可使用项目内置 `.htaccess`。

## 对象存储

后台可以添加和测试存储源：

- Local 本地测试存储
- Aliyun OSS
- Tencent COS
- Qiniu Kodo
- S3 兼容存储
- MinIO 私有云
- Cloudflare R2
- WebDAV
- BM TOOLS 同系统对接

正式运营建议使用对象存储作为默认存储源，本地存储主要用于安装后测试流程。

第三方存储接入说明：

- 阿里云 OSS：填写 AccessKey、SecretKey、Bucket、Endpoint，可选自定义访问域名。
- 腾讯云 COS：填写 SecretId、SecretKey、Bucket、Region，Endpoint 可留空由系统自动拼接。
- 七牛云 Kodo：填写 AccessKey、SecretKey、Bucket、访问域名，上传域名通常可留空。
- S3 兼容存储：填写 Access Key、Secret Key、Bucket、Region、S3 Endpoint，可对接 AWS S3、Wasabi、Backblaze B2 S3 兼容模式等。
- MinIO 私有云：填写 Access Key、Secret Key、Bucket、Region 和 MinIO S3 API Endpoint，例如 `https://minio.example.com` 或 `http://10.0.0.8:9000`，不要填写 Console 地址。
- Cloudflare R2：填写 Access Key、Secret Key、Bucket、账号级 Endpoint，Region 可留空或填写 `auto`。
- WebDAV：填写 WebDAV 地址、账号、密码和路径前缀。
- 保存后点击后台存储列表里的“真实测试”，系统会上传并删除一个极小测试文件，用于确认 Key、Bucket、Endpoint、上传权限和删除权限是否可用。
- SecretKey 会加密保存到数据库，后台编辑页不会回显明文；旧版本已保存的明文密钥会在进入存储管理或执行真实测试时自动加密。
- 对象存储请求启用 HTTPS 证书校验，建议服务器保持系统时间正确并安装可用 CA 证书。

### 多存储备份与迁移

后台 `资源功能管理` 下提供独立的多存储能力页面：

- 备份策略：配置主存储命中条件、文件范围、备份目标和失败策略。
- 备份任务：查看每个文件写入备份存储的状态，失败任务可以单条或批量重试。
- 历史补备：为旧文件补齐多存储备份副本，建议先小批量验证。
- 存储迁移：把文件迁到新的目标存储，支持本地可读源文件真实上传，也支持把已有成功备份一键切换为主文件记录。

存储迁移的设计原则是“明确可验证”：本地可读文件会真实上传到目标存储；目标存储已有成功备份时可以直接切换；远程源文件如果当前无法可靠读取，系统会跳过并写明原因，避免出现迁移假成功。

### 告警通知通道

后台 `系统设置 -> 邮件服务` 中可以配置告警通知外送：

- 企业微信机器人 Webhook
- 钉钉机器人 Webhook，可选加签 Secret
- Telegram Bot Token + Chat ID

监控异常会先写入站内通知和邮件，再尝试外送到已启用的通道。外送失败会写入 `storage/logs`，不会阻塞监控采集和告警创建。Webhook、钉钉 Secret、Telegram Token 会加密保存，后台不回显明文。

## 开放平台应用 Token

用户中心 `API 管理` 提供开放接口接入能力，旧版总 API Token 继续兼容 PicGo、ShareX 和脚本上传。新版本建议为每个客户端或第三方业务系统单独创建“应用 Token”：

- 每个应用 Token 可以独立命名，便于区分 PicGo、ShareX、业务系统同步、远端仓储站等不同来源。
- 每个应用 Token 可以单独勾选权限范围：上传文件、读取文件列表、删除文件、读取容量。
- 每个应用 Token 可以设置独立 IP 白名单，留空表示不限制。
- 创建或重置后只展示一次明文 Token，请立即复制；离开页面后无法再次查看，只能重置。
- 后台 `API 日志` 会展示应用名称、接口、状态码、返回信息和请求 IP，方便判断第三方厂商对接时是哪一步不通。

开放接口地址：

```text
POST /api/upload
POST /api/files
POST /api/delete
POST /api/quota
```

调用时在请求头中携带：

```text
X-API-Token: 你的应用Token
```

升级旧版本后如页面提示缺少 API 应用表，请进入：

```text
后台管理系统 -> 系统维护 -> 数据库巡检
```

执行一键修复，系统会补齐 `api_apps` 表和 `api_logs.app_id` 字段。

### 同系统对接

同系统对接适合没有购买大厂对象存储的新手站长。你可以准备两套 BM TOOLS：

- A 站：主站，用户在这里上传图片和网盘文件。
- B 站：仓储站，专门接收 A 站通过 API 上传的文件。

A 站后台点击“配置同系统对接”，类型选择：

```text
同系统对接
```

核心填写：

```text
远端站点地址：https://B站域名
对接 Token：B站远端用户的 API Token
```

保存后点击“真实测试”。测试会调用 B 站的容量接口、上传接口和删除接口。

安全要求：

- Endpoint 必须是 HTTPS。
- A 站请求前会校验 B 站域名解析结果，拒绝内网、本机和保留地址。
- A 站会把远端用户 API Token 加密保存到存储配置里，后台不回显明文。
- B 站远端用户建议设置 API 白名单，只允许 A 站服务器出口 IP。
- 管理员可以在 B 站后台用户管理里直接给远端用户填写 API 白名单。
- API Token 连续错误会按 IP 临时锁定，降低爆破风险。
- 同系统对接请求会带链路计数，超过 3 跳自动中断，避免多个站点互相设置默认存储造成循环上传。

详细配置文档：

```text
docs/bmtools-remote-storage-dev.md
```

## 用户权限与游客上传

后台 `系统设置` 新增了权限总开关：

- 会员图床总开关：关闭后，登录用户不能进入图床上传。
- 会员网盘总开关：关闭后，登录用户不能进入个人网盘、分享管理和个人仓库管理。
- 游客图床上传：默认关闭；开启后，未注册访客可以在首页上传图片，只返回外链，不提供文件管理。
- 游客上传归属用户 ID：游客文件会计入该用户容量；留 0 时自动使用第一个启用的管理员账号。
- 游客单图大小和每日/IP 上传数：用于限制未注册用户滥用。

后台 `套餐管理` 仍然控制套餐级权益，例如图床、网盘、API、分享等；后台 `用户管理` 可以对单个用户设置：

- 图床继承套餐 / 强制允许 / 强制禁止
- 网盘继承套餐 / 强制允许 / 强制禁止

实际判断顺序为：全站总开关 -> 单用户覆盖权限 -> 套餐权限。游客上传独立受游客上传开关、归属账号套餐容量、单图大小和 IP 每日限额控制。

## 网站主域名

后台 `系统设置` 中可以配置 `网站主域名 / 外链域名`，例如：

```text
https://www.example.com
```

配置后，图床直链、Markdown、HTML、BBCode、API 返回地址、网盘分享链接、个人仓库链接都会优先使用这个域名生成完整外链。留空时系统会尝试使用当前访问域名。

## 个人仓库

个人仓库是网盘分享的商业化升级功能。普通分享适合单个文件临时发送，个人仓库适合用户长期维护一个“资料主页”或“资源库”，例如课程资料、设计素材、项目交付文件、客户下载中心、公开作品集等。

用户登录后进入：

```text
个人仓库
```

可以完成：

- 自定义仓库名称和介绍。
- 自定义链接，例如 `/r/my-files`。
- 创建多个独立仓库，每个仓库拥有独立链接和权限配置。
- 设置访问方式：公开访问、密码访问、私密关闭。
- 设置链接到期时间、总访问上限、总下载上限。
- 设置单 IP 每日访问/下载上限，降低恶意刷流量风险。
- 设置是否允许访客下载。
- 设置是否显示仓库主人。
- 从个人网盘挑选文件加入仓库。
- 单独设置每个文件公开或私密。
- 为每个文件填写对外说明。
- 查看访问、下载、密码正确、密码错误等日志。
- 按行为、关键词、日期筛选日志，并导出 CSV。

访客打开仓库链接后，只能看到已公开的仓库文件；私密文件不会出现在公开页面。开启密码访问后，访客需要输入密码才能浏览文件。关闭下载后，公开页面只展示文件信息，不提供下载按钮。

后台提供 `分享仓库管理`，管理员可以查看全站仓库、搜索筛选、查看访问/下载数据、下架违规仓库或恢复仓库。被平台下架的仓库，访客不能访问，用户端会显示平台处理原因。

商业运营建议：

- 可以把“个人仓库”作为高级套餐能力，限制免费用户的仓库文件数量或关闭自定义链接。
- 可以在后续版本增加仓库品牌封面、自定义 Logo、付费下载、下载次数限制、访问地域统计、客户备注、批量导入文件等能力。
- 如果面向企业客户，可以把仓库包装为“客户资料交付中心”或“团队资源门户”。

## 商业卡密

后台进入：

```text
后台 -> 卡密管理
```

可以生成两类卡密：

- 余额充值卡：用户兑换后，余额立即入账，系统自动生成充值订单、充值记录和余额流水。
- 套餐兑换卡：用户兑换后，直接切换或续期套餐，系统自动生成套餐订单。

卡密支持：

- 批次名称
- 卡号前缀
- 生成数量
- 渠道备注
- 过期时间
- 内部备注
- 停用 / 恢复未兑换卡密
- 删除未兑换卡密
- 查看兑换用户、兑换时间、订单号

安全设计：

- 卡密明文只在生成后展示一次。
- 数据库只保存卡密哈希和尾号预览。
- 已兑换卡密不能删除，保留财务和售后记录。
- 用户连续输错卡密会被临时限制，降低撞库和爆破风险。

用户进入：

```text
卡密兑换
```

输入卡密后，系统会自动判断是余额卡还是套餐卡。相同套餐未到期时，套餐卡会按当前到期时间顺延。

## 服务器监控

## 在线更新

后台 `系统维护` 页面已提供在线更新基础流程：

- 检查远程更新清单
- 下载 ZIP 更新包
- 校验更新包 SHA-256
- 安装前自动备份当前数据库
- 安装前自动备份即将被覆盖的文件
- 支持更新包内声明 SQL 升级文件
- 安装完成后写入当前版本和最后更新时间

更新清单示例：

```json
{
  "version": "1.0.1",
  "released_at": "2026-05-12 12:00:00",
  "min_php": "7.4.0",
  "package_url": "https://example.com/bmtools/bmtools-1.0.1.zip",
  "package_sha256": "更新包 SHA-256",
  "changelog": [
    "新增在线更新流程",
    "优化系统维护中心"
  ]
}
```

更新包可以在根目录放置可选的 `update.json`：

```json
{
  "version": "1.0.1",
  "min_php": "7.4.0",
  "sql": [
    "database/updates/20260512_update.sql"
  ],
  "delete": []
}
```

安全边界：

- ZIP 根目录应直接对应项目根目录，例如 `app/Controllers/...`、`public/assets/...`。
- 更新包只能写入 `app`、`public`、`database`、`config/app.php`、根目录入口文件和文档文件等项目路径。
- 禁止覆盖 `storage`、`public/uploads`、`config/database.php`、`.env`。
- 远程更新地址仅允许 `http` / `https`，并拒绝本机、内网和保留 IP。
- SQL 文件中的 `__PREFIX__` 会自动替换为当前数据库表前缀。

- 后台需要管理员登录。
- 密码使用哈希存储。
- 登录失败会按 IP 和账号做频率限制，达到上限后临时锁定，降低爆破风险。
- 邮箱验证码按邮箱和 IP 限频，避免验证码轰炸。
- 个人仓库密码访问会按访客 IP 限制错误尝试次数。
- API Token 缺失或错误会按调用 IP 限制尝试次数，降低接口 Token 爆破风险。
- 用户 API Token 会加密保存，并额外保存 SHA-256 摘要用于接口检索；旧版本明文 Token 会在访问个人资料、API 管理或接口调用时自动升级。
- 对象存储 SecretKey、SMTP 邮件密码等敏感配置会加密保存，后台不回显明文。
- IP 判断默认以 `REMOTE_ADDR` 为准，只有请求来自本机或内网代理时才读取转发头，避免访客伪造 `X-Forwarded-For` 绕过白名单或限频。
- 记住登录 Token 在数据库中只保存 SHA-256 摘要，避免数据库泄露后直接复用登录态。
- 会话 Cookie 和记住登录 Cookie 使用 HttpOnly、SameSite=Lax，并在 HTTPS 下自动启用 Secure。
- 全站响应增加基础安全头：X-Frame-Options、X-Content-Type-Options、Referrer-Policy、Permissions-Policy，并移除 X-Powered-By。
- `public/uploads/.htaccess` 已禁止 Apache 环境在上传目录执行 PHP/CGI 等脚本；Nginx 环境也建议在站点配置里禁止上传目录脚本执行。
- 重要表单使用 CSRF 校验。
- 支付回调需要验签。
- API 使用 Token 鉴权。
- 管理员操作会写入日志。
- 系统维护功能只在后台展示，不提供隐藏入口。

## 2026-6.1日功能更新能力补充

### 网站主域名 / 外链域名

后台进入：

```text
后台 -> 系统设置 -> 网站主域名 / 外链域名
```

填写示例：

```text
https://www.example.com
```

保存后，图床直链、Markdown、HTML、BBCode、API 返回地址、网盘普通分享、个人仓库分享都会优先使用这个域名生成完整外链。留空时，系统会尽量使用当前访问域名自动生成完整链接。

### 普通网盘分享

普通分享适合单个文件快速发给别人。分享记录页面已经支持：

- 显示完整分享链接。
- 一键复制当前分享链接。
- 自动显示分享二维码。
- 统计访问次数和下载次数。
- 记录访问、下载、提取码错误、过期、拦截等日志。

访问/下载显示格式：

```text
访问次数 / 下载次数 / 下载上限
```

如果没有下载上限，只显示：

```text
访问次数 / 下载次数
```

### 个人专属分享仓库

个人仓库适合长期维护资源合集，适合课程资料、设计素材、客户交付文件、自媒体粉丝资料包等场景。当前支持：

- 多仓库创建。
- 自定义 `/r/xxx` 仓库链接。
- 公开、密码、私密三种访问模式。
- 仓库封面图 URL。
- 仓库标签。
- 分享文案一键复制。
- 仓库二维码。
- 公开页展示封面、标签、统计卡片。
- 图片文件在公开页直接预览。
- 单文件公开/私密控制。
- 仓库文件批量加入公开、批量加入私密、批量移除。
- 公开仓库支持本地存储文件打包下载 ZIP。
- 访问日志、下载日志、密码错误日志、风控拦截日志。
- CSV 日志导出。

运营建议：普通分享用于临时单文件分发，个人仓库用于长期资源运营和会员权益包装。
